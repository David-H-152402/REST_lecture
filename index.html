What is REST?

What is REST?

REST is an architectural style of APIs, where the focus is on a specific set of interactions between the client and server, rather than implementation details. Its purpose is to induce performance, scalability, simplicity, modifiability, visibility, portability, and reliability.

More about REST

  • REpresentational State Transfer - described by Roy Fielding in his doctoral dissertation in 2000
  • A service based on REST is called a RESTful service
  • Not dependent on any protocol, but almost every RESTful service uses HTTP

What are the 5 Constraints of REST?

5 Constraints of REST

  1. Client-Server
  2. Stateless
  3. Cacheable
  4. Uniform Interface
  5. Layered System
  6. Code on Demand

What are the 5 RESTful routes of CRUD?

The 5 RESTful Routes

  • app.get('/') - LIST - Get all
  • app.get('/:id') - READ - Get one
  • app.post('/') - CREATE - Post one
  • app.put('/:id') - UPDATE - Update one
  • app.delete('/:id') - DELETE - Delete one

What is the difference between server-side rendering and decoupled apps?

Server-side Rendering vs Decoupled Apps

  • Server-side rendering means everything is served from the server - there is no client
  • Server-side means tightly coupled
  • Usually have 7 routes (not 5)

What are idempotent methods?

Safe and Indempotent Methods

  • Safe operations do not have any effect on the original value of the resource
  • Idempotent operations give the same result no matter how many times you perform it
